# 总纲: LLM注意力模型与提示词结构优化
# 核心原则: LLM对完整提示词的注意力分布呈U形,即开头和结尾部分的注意力权重最高,中间部分最低
# 指导方向: 必须依据此U形注意力模型,对提示词中不同信息的布局进行策略性优化,以实现最大化指令效能

# 分论点一: U形注意力模型的定义与应用
# 定义:
#   - 在模型接收到的完整上下文(prompt)中,信息的重要性由其位置决定
#   - 最高注意力区域: 上下文的起始位置 (prompt anfang) 和结束位置 (prompt ende)
#   - 最低注意力区域: 上下文的中间位置
# 应用原则:
#   - 将最关键的指令与信息,分别置于上下文的开头与结尾,形成双端强调布局
#   - 将次要或辅助性信息,置于上下文的中间区域

# 分论点二: 基于U形注意力的世界书(World Info)布局策略
# 目标: 将不同性质的世界书条目,精确映射到U形注意力的不同区域
# 布局指南:
#   - 1. 开头区域 (最高注意力):
#     - 适用内容: 基础性/全局性/恒定不变的核心设定 (例如: 世界观基本法/角色核心背景)
#     - 实现方式: 使用"角色定义之前世界书"类型,并设定为最高优先级 (即最小顺序数字),确保其在整个上下文的最顶部
#   - 2. 结尾区域 (最高注意力):
#     - 适用内容: 动态/即时/具备最高优先级的指令 (例如: "破限"类指令/当前场景的关键变量)
#     - 实现方式: 使用确保能置于上下文最末端的功能 (例如: "At Depth"类型的世界书条目)
#   - 3. 中间区域 (最低注意力):
#     - 适用内容: 重要性较低/补充性/情景关联性较弱的信息
#     - 风险认知: 角色卡&"角色定义之后世界书"&历史消息的长度,共同决定了哪些内容会被挤压至注意力最低的中间区域。必须对提示词的最终结构有清晰的预估

# 分论点三: 标准化操作指南
# 指导一: 双端配置原则
#   - 任务: 将关键信息系统性地分配到开头和结尾两个高注意力区域
#   - 标准: 评估所有需要注入的信息,根据其"重要性"和"恒定性"两个维度,将其归类至"开头适用"或"结尾适用"类别
# 指导二: 信息分层原则
#   - 任务: 对信息进行优先级排序与分层
#   - 标准:
#     - P0 (最高优先级): 必须被模型注意到的指令 -> 置于开头或结尾
#     - P1 (次高优先级): 强相关设定 -> 置于开头或结尾的次要位置
#     - P2 (普通优先级): 补充性信息 -> 允许置于中间区域
# 指导三: 预设模板一致性原则
#   - 任务: 确保预设(prompt template)的结构稳定,避免引入过长的静态内容
#   - 标准: 预设中,置于世界书或历史消息前后的静态文本(如系统提示)应保持简洁。过长的静态文本会改变实际上的"开头"与"结尾"位置,导致U形注意力模型应用失效
</note>
